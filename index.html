Fantastic Roadmaps

HTML

É a linguagem de marcação usada para estruturar o conteúdo de páginas web, definindo elementos como textos, imagens e links.

Alguns de seus principais conceitos:

Tags, Elementos, Atributos e Semântica.

CSS

É a linguagem de estilos responsável por definir a aparência e o layout das páginas, permitindo personalização e responsividade (adaptação do website para diferentes tamanhos de tela).

Alguns de seus principais conceitos:

Selectors, Box Model, Flexbox, Grid, Media Queries

JavaScript

É uma linguagem de programação que adiciona interatividade ao site, permitindo manipulação do DOM e execução de lógica no navegador.

Alguns de seus principais conceitos:

Variáveis, Funções, Eventos, DOM, Arrays, Objetos.

React

É uma biblioteca JavaScript para criar interfaces dinâmicas e reutilizáveis, baseada no conceito de componentes e estado.

Alguns de seus principais conceitos:

Componentes, Props, State, Hooks.

TypeScript

É uma extensão do JavaScript que adiciona tipagem estática, tornando o código mais seguro e fácil de manter.

Alguns de seus principais conceitos:

Types, Interfaces, Generics, Enums.

Next.js

É um framework para React que otimiza o desenvolvimento de aplicações web, oferecendo renderização híbrida e melhorias de performance.

Alguns de seus principais conceitos:

Páginas, SSR, API Routes, Static Generation.

FAQ

Por que seguir a ordem recomendada?
A ordem recomendada (1 - HTML, 2 - CSS, 3 - JavaScript, 4 - React, 5 - TypeScript, 6 - Next.js) faz sentido porque cada tecnologia depende da anterior:

HTML é a base estrutural da web.

CSS é responsável pela estilização e aparência.

JavaScript adiciona interatividade e lógica.

React depende de JavaScript para criar componentes reutilizáveis.

TypeScript melhora o JavaScript com tipagem estática, útil para projetos grandes.

Next.js é um framework baseado em React que otimiza a performance e a organização do código.

Seguir essa ordem ajuda a entender cada etapa e evita frustrações ao lidar com conceitos avançados antes da base.

Posso iniciar por React, TypeScript ou Next.js?
Tecnicamente, sim, mas não é recomendado.

React e Next.js exigem conhecimento prévio de HTML, CSS e JavaScript, pois são baseados nessas tecnologias.

TypeScript adiciona complexidade extra ao JavaScript, podendo dificultar o aprendizado se você ainda não dominar a linguagem base.

Se você já tem experiência com programação, pode tentar aprender React diretamente, mas entender bem HTML, CSS e JavaScript primeiro tornará o aprendizado mais fluido.
Qual a diferença entre biblioteca e framework?

Uma biblioteca (exemplo: React) é um conjunto de funções que você pode usar livremente no seu código sem seguir uma estrutura rígida. Você decide como organizá-la no projeto.

Um framework (exemplo: Next.js) já impõe uma estrutura e regras para o desenvolvimento, oferecendo mais funcionalidades prontas e padronizadas.

Em resumo, bibliotecas dão liberdade, enquanto frameworks fornecem uma estrutura mais definida.
Quais exemplos de projetos são recomendados para iniciantes?
Aqui estão algumas ideias, organizadas por nível de dificuldade:

HTML + CSS: Página de portfólio, blog simples, landing page responsiva.

JavaScript: To-do list, calculadora, relógio digital, galeria de imagens interativa.

React: Lista de tarefas dinâmica, app de previsão do tempo com API, quiz interativo.

TypeScript: Aplicação CRUD (criar, ler, atualizar, deletar dados), agenda de contatos.

Next.js: Blog com CMS, e-commerce básico, dashboard de estatísticas.

Cada um desses projetos ajuda a reforçar conceitos fundamentais antes de avançar para algo mais complexo.
